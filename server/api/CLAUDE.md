# Règles — server/api/

**OBLIGATOIRE** : Après toute création ou modification d'un endpoint dans ce répertoire, mettre à jour `@api-docs.md` à la racine du projet.

Pour chaque endpoint documenter :

- Méthode + chemin (ex: `GET /api/auth/session`)
- Description courte
- Paramètres / body attendus
- Réponse retournée
- Codes d'erreur possibles
